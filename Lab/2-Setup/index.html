    Prerequisites and Setup

    SIMPLIFIED SETUP WITH IBM TECHNOLOGY ZONE

    As of March 11, 2024, previous versions of the Technology Preview have been discontinued by IBM and Red Hat.

    Via the IBM Technology Zone, IBM technical sellers and business partners can reserve a "Jumpbox" Linux virtual machine (VM) with VS Code and IBM watsonx Code Assistant for Red Hat Ansible Lightspeed pre-installed. Minor setup and replication of sample code is still required, but using this environment will simplify many of the following prerequisite steps and potentially eliminate any technical hiccups that may occur along the way. It is the recommended path for getting started with the hands-on lab material.

    Before getting started with IBM watsonx Code Assistant for Red Hat Ansible Lightspeed (WCA), an environment must be reserved and deployed via the IBM Technology Zone (ITZ).

    These environments be reserved for learning and client demonstration purposes at no-charge. Some further configuration of these environments will be required after deployment.

    Red Hat Training and Red Hat Connect

    IBM employees and business partners will need to request access to Red Hat Connect.

    Enrollment in a Red Hat Connect account is freely available to IBM and Red Hat employees, as well as IBM business partners.

    • If you are an IBM employee: https://ibm.seismic.com/Link/Content/DCblzS-

    • If you are an IBM business partner, and need to apply for a Red Hat partnership, select from the Technology or Business partner route for your organization: https://connect.redhat.com/sites/default/files/pdf/Applying_for_Red_Hat_Partnership.pdf

    • For all support inquiries related to Red Hat Connect accounts, consult the following resources: https://connect.redhat.com/support

    Request an IBM Technology Zone environment

    You will require access to the ITZ in order to reserve your environment and complete the lab. If you do not yet have access or an account with the ITZ, you will need to register for onea>.

    1. Log in to the ITZ and click the WCA For Ansible Essentials Plan: Visual Studio Desktop 1.2 tile from the Environment options.

      URL: https://techzone.ibm.com/collection/ibm-watson-x-code-assistant-for-ansible-lightspeed/environments


    1. From the Single environment reservation options, select Reserve now.


    1. Supply additional details about your ITZ reservation request:

      • Name: Give your reservation a unique name.

      • Purpose: Set to Practice / Self-Education and affirm that customer data will not be used with the environment. If you are replicating this hands-on demonstration with a client, you must select Customer Demo and supply a sales opportunity number.

      • Purpose Description: Provide a brief summary of how the environment will be used.

      • Preferred Geography: Select the data center region that is closest to your location.

      • End Date & Time: Select a time and date for when the reservation will expire. The recommended amount is 2 days, although it is possible to finish the hands-on demonstration within a few hours. Additional time extensions are available.

      • VPN Access: Set to Disable

      When satisfied, verify that you agree to the Terms and Conditions for the environment and finalize your reservation request by clicking Submit.

      PROVISIONING TIMES

      Reservations take approximately 15-30 minutes to complete from the time that you click submit. If you navigate to the My Reservations tab of the ITZ, you can monitor the progress of your reservation. Wait for the ITZ reservation to be marked as "Ready" before attempting to start the lab.


    1. When the ITZ environment is ready to go, you will receive an email to your inbox (similar to the screenshot below). Click the blue View My Reservations at the bottom of the email to open a page with more details about the environment.

      Alternatively, you can click the My Reservations tab from the ITZ home page to drill down into your environment's details.


    1. Near the top of the page, under Published Services, click the noVNC link to access the virtual machine.


    1. A new tab will load - click the Connect button to continue with authentication.


    1. Supply the following password: IBMDemo0s!

    1. Once you have successfully authenticated, look for the VS Code application shortcut at the bottom of the desktop and click it to start up the service.

      VS (Visual Studio) Code is a code editor built on open source technology that is free to use and published by Microsoft. Distributions are available for both Windows and macOS operating systems.


    1. VS Code will load and then present you with an Authentication Required splash screen.

      • Click the Cancel button. You may need to do this two times.

      • Wait until the VS Code application loads before continuing.


    1. From the left side of the interface, click the large A tab — this corresponds to the Red Hat Ansible extension, which has been pre-installed for this VS Code environment.


    1. Ansible Lightspeed requires that users re-authenticate with Red Hat before its capabilities are activated inside the VS Code environment.

      • Click the blue Connect button from the left-hand side of the interface

      • When prompted, click Allow

      • When asked Do you want Code to open the external website?, click Open


    1. Once again, the Authentication Required splash screen will appear.

      As before, click the Cancel button. You may need to repeat this action to fully dismiss the warning.


    1. Google Chrome's web browser will open and begin loading.

      • Dismiss any warnings regarding Can't update Chrome by clicking the grey X button

      • If the page appears to "hang" and not load, click the Stop and Refresh button (as shown below), just to the left of the URL address bar

      • Once the page has loaded, click the Log in with Red Hat button


    1. When prompted to Log in to your Red Hat account, enter the login or email address associated with your personal Red Hat Connect account. As a reminder, instructions for how to sign up are available at the top of this page.

      • Click Next to continue

      • Enter the password for your account and click Log in


    1. You will be prompted to provide a little more information about yourself. Select whatever option you feel appropriate from the Job Title field (the only mandatory field on the form) and then click Submit at the bottom of the page to continue.

    1. An Authorize Ansible Lightspeed for VS Code splash screen will load.

      Click Authorize to continue.


    1. Chrome's web browser will close and the VM will be returned to the VS Code application. Look for a confirmation that you are logged in, which should appear in both the bottom-right of the screen as a pop-up and the top-left panel of the interface.

      UNABLE TO LOGIN

      If too much time was spent on the initial login attempt, you may seem a time-out warning along the bottom of the screen that reads Cancelling the Ansible Lightspeed login after 60s. Try again.

      In that case, click the blue Connect button once again as re-authenticate in the same way that you did for Steps 11-16 previously. The login experience should be much smoother (and faster) on the second attempt.



    1. With the Ansible Lightspeed extension authenticated and active, click the Explorer button (as shown) in the top-left corner of the interface.

      Confirm that the working directory is set to ANSIBLE-WCA-DEMO-KIT and that it contains files similar to those listed in the screenshot below.


    1. As a final step, verify that the VM has access to the latest demo code by performing the following git fetch and git pull commands:

      • From the top of the VM interface, drill down into View and then Command Palette

      • Into the pop-up console, type git fetch and then hit Return — wait for the operation to finish

      • Open the console once again, type git pull, and then hit Return

    At this stage, the hands-on environment has been fully configured.
